Encrypted Data API

REST methods that can be used to store data in the hosted server and retrieve the encrypted data at a later time.

POST New values with server created keys
https://dev68039.service-now.com/api/x_165574_encrypt/data_encrypter_api?key_type=server

Example POST that will generate a random key from the server for each value. - API always expects an array consisting of strings and/or objects. Items in the array can be constructed in multiple ways: - A string value - an object with a single "property name":"this is the value" property. (The "property name will be ignored") - an object with a designated "value" property (other non-id properties are ignored) Objects optionally have an "id" property for your system to reference later, otherwise the "id" value will match the index of the object as it appears in the original array.

Optional. Defaults to false if not provided. If set to true, will provide a full response as if the records were created, but will not actually record the values to the database.

POST Example of what occurs when the same id is used in the same request
https://dev68039.service-now.com/api/x_165574_encrypt/data_encrypter_api?key_type=client

Objects can optionally have the "id" property. When provided, the response will name matching responses accordingly for your convenience. If no id is provided, the id is set to the index of where that object was in the provided array. If the same id is used for multiple objects, the first successful object will be given the id as named, all subsequent objects will have the index of where that object was in the provided array appended to the id.

GET Retrieve values
https://dev68039.service-now.com/api/x_165574_encrypt/data_encrypter_api?key=Alpha&key=Charlie&key=Echo235c2281db5adc100fbc87423996199d&delete_keys=true

Method to retrieve values based on keys provided. For every key parameter provided in the query, the corresponding value will be returned. Be aware that values will be URL decoded. Eg. "Hello%20World" will be interpretted by the API as "Hello World"

Assumed

None

true

Optional. If not provided, defaults to false. If provided, must be set to \"true\". Will delete the keys after the transaction ends.

GET Retrieve values - alternative
https://dev68039.service-now.com/api/x_165574_encrypt/data_encrypter_api?key=["Alpha","Charlie","Echo"]

Alternative method to retrieve values based on keys provided. Instead of providing each key as a separate parameter, JSON.stringify an array of strings and set it to one key parameter. Be aware that values will be URL decoded. Eg. "Hello%20World" will be interpretted by the API as "Hello World"

HEADERS
Accept
text/plain

Assumed

Content-Type
text/plain

Assumed

PARAMS
key
[\"Alpha\",\"Charlie\",\"Echo\"]

Required. Must be a JSON.stringify array of strings. If more than one key parameter is provided, this method is ignored and the standard method is used.

delete_keys
true

Optional. If not provided, defaults to false. If provided, must be set to \"true\". Will delete the keys after the transaction ends.

Example Request
  • Retrieve values - alternative
                                                
GET https://dev68039.service-now.com/api/x_165574_encrypt/data_encrypter_api?key=["Alpha","Charlie","Echo"]
